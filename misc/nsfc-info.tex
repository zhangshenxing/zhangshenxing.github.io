\nsfcb{参照以下提纲撰写，要求内容翔实、清晰，层次分明，标题突出。}\nsfc{\bfseries 请勿删除或改动下述提纲标题及括号中的文字。}

\nsfcsec{{\bfseries（一）立项依据与研究内容}（{\bfseries 建议 8000 字以下}）：}

\nsfcsec{1．{\bfseries 项目的立项依据}（研究意义、国内外研究现状及发展动态分析，需结合科学研究发展趋势来论述科学意义；或结合国民经济和社会发展中迫切需要解决的关键科技问题来论述其应用前景。附主要参考文献目录）；}

\alert{立项依据}回答为何要做的问题。
类似于论文的Introduction，很重要。
\begin{itemize}
	\thinline
	\item 介绍研究领域和意义
	\item 逐步聚焦到具体研究方向（国内外研究趋势和现状，关键参考文献）
	\item 提出尚未解决的具体科学问题（卡在哪里）
	\item 简要阐述解决方案和预期贡献（我来搞定）
	\item 关键句可highlight，可插入2-3幅图
\end{itemize}

\alert{常用表述}
\begin{itemize}\kaishu
	\thinline
	\item ……具有重要的科学意义。……已经成为国际……研究的热点前沿方向
	\item 虽然前人研究……，但是……，这严重阻碍了……
	\item 鉴于此，本项目计划开展……，重点探索……。本项目的研究结果预期将解决……，揭示……，显著促进……
\end{itemize}

\alert{参考文献}
\begin{itemize}
	\thinline
	\item 30条左右为宜
	\item 国际为主，兼顾国内
	\item 近几年为主
	\item 适当引用自己论文
	\item 格式严谨
\end{itemize}

\alert{常见问题}
\begin{itemize}
	\thinline
	\item 对研究背景只是泛泛描述文献报道，缺乏深入分析
	\item 直奔主题，缺少Zoomin过程
	\item 思路不清晰，文气不连贯
	\item “填补空白”、“首次”
	\item 缺失关键参考文献(甚至学术不端)
	\item 参考文献格式不规范
\end{itemize}



\nsfcsec{2．{\bfseries 项目的研究内容、研究目标，以及拟解决的关键科学问题}（此部分为重点阐述内容）；}

\alert{研究内容}回答做什么的问题。
\begin{itemize}
	\thinline
	\item 紧密围绕拟解决的科学问题和研究目标，不宜分散
	\item 符合项目经费能支撑的体量，不宜过多(最多4项)
	\item 每项研究内容标题之后要有具体说明
\end{itemize}

{\CJKfamily{afbkaishu}\enafbkaishu
本项目的主要研究内容是测量……，计算……，考察解析……，构建……。具体包括以下三个方面
\begin{enumerate}[label=\arabic*.]
	\thinline
	\item 标题 内容
	\item 标题 内容
	\item 标题 内容
\end{enumerate}
}

\alert{研究目标}回答做了有什么用的问题。
应是一段文字，不要分项目标集中、明确，不宜过多可以写在研究内容之前
本项目的研究目标是建立……方法，确定.性质参数，查明……规律，建立……模型，揭示/阐明……原因/机制，促进……认识，发展……理论

\alert{拟解决的关键科学问题}
\begin{itemize}
	\thinline
	\item 核心科学问题的分解
	\item 可分项表列，不直过多(最多3项)
	\item 与研究目标有联系，但表述方式不同不一定要用问句
\end{itemize}
例如:
\begin{itemize}
	\thinline\CJKfamily{afbkaishu}\enafbkaishu
	\item 俯冲板片流体迁移碳的能力(有多强?)
	\item 碳在流体中的赋存形式(是什么?)
	\item 地球深部碳循环过程和机制(如何?)
\end{itemize}



\nsfcsec{3．{\bfseries 拟采取的研究方案及可行性分析}（包括研究方法、技术路线、实验手段、关键技术等说明）；}

\alert{研究方案、年度研究计划}回答怎么做的问题。
\begin{itemize}
	\thinline
	\item 对科学问题的解决思路
	\item 对研究内容的具体说明
	\item 可使用流程图、计划表
	\item 把握好分寸(自我保护)
\end{itemize}

\alert{常见问题}
\begin{itemize}
	\thinline
	\item 过于简单:只有方法名称，未阐述关键环节
	\item 过于详细:大篇幅阐述常规方法
	\item 可行性差
\end{itemize}

``研究目标$\rightarrow$关键科学问题$\rightarrow$研究内容$\rightarrow$研究方案$\rightarrow$年度研究计划''应当是一个逐步加细的过程。

\alert{可行性分析}
\begin{itemize}
	\thinline
	\item 主要指研究思路和研究方案的可行性
	\item 有的人也把研究基础和工作条件加进去
	\item 为什么你觉得能做成?
	\item 可能出现哪些问题，如果出现有无解决办法?(风险分析)
	\item 要与创新性相协调
\end{itemize}


\nsfcsec{4．{\bfseries 本项目的特色与创新之处；}}


很重要，强调与众不同、独具匠心之处
\begin{itemize}
	\thinline
	\item 科学问题创新
	\item 研究思路创新
	\item 技术方法创新
	\item 多种手段综合研究
	\item 不宜过多(最多4点)
	\item 把握好度，表述过犹不及(首次、填补空白、国际领先，……)
\end{itemize}




\nsfcsec{5．{\bfseries 年度研究计划及预期研究结果}（包括拟组织的重要学术交流活动、国际合作与交流计划等）。}

\alert{预期研究结果}是指能做到什么程度。
\begin{itemize}
	\thinline
	\item 研究成果: 与研究目标、关键科学问题相关，但可以更具体可预期、可量化、可检查
	\item 成果表现形式: 论文、专利
	\item 人才培养: 研究生
\end{itemize}
